Bio

Penghui Li is a Postdoctoral Research Scientist at Columbia University. His research focuses on software system security and and its intersection with software engineering, programming languages, and machine learning. He develops rigorous program analysis techniques and autonomous LLM agents to address critical security challenges such as vulnerability detection, exploit generation, and runtime defense. Penghui has discovered over 300 security vulnerabilities, leading to patches in foundational systems such as the Linux kernel, PHP interpreter, and GitHub platform. Penghui's work has been recognized with a Distinguished Paper Award, a Distinguished Artifact Award, and a Best Paper Honorable Mention at ACM CCS. He also received two Distinguished Reviewer Awards.

  • Holistic Concolic Execution for Dynamic Web Applications via Symbolic Interpreter Analysis [PDF]
    Penghui Li, Wei Meng, Mingxue Zhang, Chenlin Wang, Changhua Luo
    In Proceedings of the 45th IEEE Symposium on Security and Privacy (S&P), May 2024
    ▹ This paper presents a new approach that symbolically analyzes the language interpreter to solve the long-standing problem of incomplete syntax support in symbolic execution. It reduces the engineering effort from over a year to under two weeks.

  • FuzzCache: Optimizing Web Application Fuzzing Through Software-Based Data Cache [PDF]
    Penghui Li, Mingxue Zhang
    In Proceedings of the 31st ACM Conference on Computer and Communications Security (CCS), October 2024
    ★ Distinguished Paper Award
    ▹ This paper introduces a systematic optimization for dynamic analysis of web applications that eliminates expensive, repetitive data access. The solution is generic and can complement all existing dynamic analysis tools by boosting testing throughput by 4 times.

  • SDFuzz: Target States Driven Directed Fuzzing [PDF]
    Penghui Li, Wei Meng, Chao Zhang
    In Proceedings of the 33rd USENIX Security Symposium (Security), August 2024
    ▹ This paper presents a directed fuzzing technique that guides testing toward target program states rather than just code locations. By incorporating both control-flow and data-flow information, it efficiently reaches deep, complex bugs and validates suspected vulnerabilities.
