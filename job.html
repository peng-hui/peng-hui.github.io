Bio

Penghui Li is a Postdoctoral Research Scientist at Columbia University. His research focuses on software security and privacy, with work that spans software engineering, programming languages, and machine learning. He develops rigorous program analysis techniques and autonomous LLM agents to address critical security challenges, including vulnerability detection, exploit generation, and runtime defense. Penghui has discovered over 300 security vulnerabilities, leading to critical patches in foundational systems such as the Linux kernel, PHP interpreter, and GitHub platform. His research has been recognized with a Distinguished Paper Award, a Best Paper Honorable Mention, and a Distiguished Artifact Award at ACM CCS. He has also received the CCS Top Reviewer Award and USENIX Security Distinguished Artifact Reviewer Award.

Representative Works [Full List]

  • Holistic Concolic Execution for Dynamic Web Applications via Symbolic Interpreter Analysis [PDF]
    Penghui Li, Wei Meng, Mingxue Zhang, Chenlin Wang, Changhua Luo
    In Proceedings of the 45th IEEE Symposium on Security and Privacy (S&P), May 2024
    ▹ This paper presents a new approach that symbolically analyzes the language interpreter to solve the long-standing problem of incomplete syntax support in symbolic execution. It reduces the engineering effort from over a year to under two weeks.

  • FuzzCache: Optimizing Web Application Fuzzing Through Software-Based Data Cache [PDF]
    Penghui Li, Mingxue Zhang
    In Proceedings of the 31st ACM Conference on Computer and Communications Security (CCS), October 2024
    ★ Distinguished Paper Award
    ▹ This paper introduces a systematic optimization for dynamic analysis of web applications that eliminates expensive, repetitive data access. The tool is generic and can complement all existing dynamic analysis tools by boosting testing throughput by 4 times.

  • SEDiff: Scope-Aware Differential Fuzzing to Test Internal Function Models in Symbolic Execution [PDF]
    Penghui Li, Wei Meng, Kangjie Lu
    In Proceedings of the 30th ACM Joint European Software Engineering Conference and Symposium on the Foundations of Software Engineering (ESEC/FSE), November 2022
    ▹ This paper introduces the first tool for testing the correctness of symbolic execution engines. It reveals that even commonly used engines contain critical bugs in their internal function models that can lead to unsound analysis results.
